# 从 v1.0 升级到 1.1

本文说明如何从 **workOrder v1.0**（固定四状态、等待对象/原因写死在数据库列）升级到 **v1.1**（可配置「代办状态」与自定义字段）。

---

## 升级会做什么

首次启动 v1.1 时（或使用迁移工具时），程序会自动：

| 步骤 | 说明 |
|------|------|
| 数据库列 `extra_fields` | 若不存在则添加 |
| 等待字段迁移 | 将 `waiting_for` / `waiting_reason` 写入 `extra_fields` JSON |
| `status_config.json` | 若不存在则生成（等价于 v1.0 的四状态 + 待回复字段） |
| 附件目录 | 不变，`attachments/` 原样保留 |

**不会**修改已有工单的 `status` 值；**不会**覆盖已有的 `status_config.json`。

---

## 方式一：覆盖升级（推荐）

最简单，适合大多数用户。

### 安装版（setup.exe / msi）

1. 在 v1.0 中打开 **设置 → 数据备份**，导出 ZIP（推荐）
2. **完全退出** workOrder（任务管理器确认无 `workOrder.exe`）
3. 运行 v1.1 安装程序，安装到 **原路径**
4. 启动 v1.1

### 便携版

1. （推荐）在 v1.0 中备份，或复制整个 `data/` 文件夹
2. 退出 v1.0
3. 用 v1.1 的 `workOrder.exe` **覆盖**旧 exe
4. **保留** `data/`、`settings.json`（如有）
5. 双击 `workOrder.exe` 或 `启动 workOrder.bat`

```
portable/
├── workOrder.exe          ← 替换为 v1.1
├── workOrder-migrate.exe  ← v1.1 新增（可选）
├── 启动 workOrder.bat
├── 迁移数据.bat           ← v1.1 新增（可选）
├── settings.json          ← 保留
└── data/                  ← 保留（含 workorder.db、attachments/）
    ├── workorder.db
    ├── attachments/
    └── status_config.json ← 首次启动后自动生成
```

---

## 方式二：v1.0 备份 ZIP 恢复到 v1.1

1. 在 v1.0 导出备份 ZIP
2. 安装/解压 v1.1
3. v1.1 中 **设置 → 恢复...**，选择 v1.0 的 ZIP
4. 按提示重启

旧 ZIP 不含 `status_config.json` 是正常的；重启后会自动生成默认配置，工单与附件完整保留。

---

## 方式三：迁移工具（可选）

在 **不启动主程序** 的情况下先迁移数据：

```bat
cd portable
迁移数据.bat
```

或命令行：

```bat
workOrder-migrate.exe --data-dir .\data
```

开发者环境：

```bash
cd src-tauri
cargo run --bin workorder-migrate -- --data-dir ../data
```

## 升级后验证

- [ ] 旧代办列表、标题、状态显示正常
- [ ] 原「待回复」工单的 **等待对象 / 等待原因** 在详情中可见
- [ ] **设置 → 代办状态** 可看到默认四状态
- [ ] 处置过程、附件图片完整
- [ ] （若用过备份）设置 → 恢复 仍可用

## v1.1 界面变化（预期行为）

| v1.0 | v1.1 |
|------|------|
| 列表「显示已完成」勾选 | 已移除；用状态多选筛选 |
| 已完成不参与逾期标红 | 所有有截止日期的过期代办均标红 |
| 切到待回复自动写进度 | 已移除 |
| 状态固定四种 | **设置 → 代办状态** 可自定义 |

## 回退到 v1.0

1. 保留升级前的 `workorder.db` 备份
2. 装回 v1.0 的 exe
3. 用备份还原 `data/workorder.db`

若在 v1.1 中修改过「代办状态」配置，回退后 v1.0 会忽略 `status_config.json` 和 `extra_fields`，但 `waiting_for` 列在迁移时未删除，一般仍可正常打开。

## 常见问题

**Q：必须运行 `迁移数据.bat` 吗？**  
A：不必。直接启动 v1.1 会自动执行相同迁移。

**Q：数据目录不在默认 `data/` 怎么办？**  
A：在 v1.0 设置页查看路径；迁移工具指定 `--data-dir "你的路径"`。

**Q：开发版如何从 v1.0 portable 迁过来？**  
A：复制 v1.0 的 `data/`（及可选 `settings.json`）到项目根或便携目录，启动 `npm run tauri dev` 即可自动迁移。
