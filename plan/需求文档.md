# workOrder 个人代办清单 — 需求文档

## 1. 项目背景

### 1.1 项目目标

workOrder 是一款面向 **Java 后端开发者** 的个人工作事项追踪工具。核心目标是：

- 将日常开发、线上问题等工作事项以清单形式罗列
- 追踪每项事项的处置进度，防止遗漏
- 随时清楚「当前还需要完成什么」

### 1.2 使用场景

- **日常个人使用**：随时记录和查看当前待办
- **混合场景**：日常开发任务 + 线上问题 / 紧急事项并行管理

### 1.3 用户角色

- 单一用户（个人工具，无多用户 / 协作需求）

## 2. 功能需求

### 2.1 脑图原有功能

#### 2.1.1 列举代办

| 编号 | 需求 | 说明 |
|------|------|------|
| F-001 | 创建代办事项 | 用户可新增一条工作事项 |
| F-002 | 展示代办列表 | 以列表形式展示所有事项 |
| F-003 | 展示处置状态 | 每条事项显示当前状态 |
| F-004 | 展示时间信息 | 展示创建时间、最后更新时间 |

#### 2.1.2 追踪代办

| 编号 | 需求 | 说明 |
|------|------|------|
| F-005 | 记录处置过程 | 支持追加、编辑、删除过程日志，按时间线展示 |
| F-006 | 更新处置状态 | 可随时变更事项状态 |

#### 2.1.3 状态标签

| 编号 | 需求 | 说明 |
|------|------|------|
| F-007 | 内置四种状态 | 未处置、处置中、待回复、已完成 |
| F-008 | 状态单选 | MVP 阶段每条事项仅一个状态 |

> 脑图中的「自定义 + 多选组合」暂不纳入 MVP，列入后续版本。

#### 2.1.4 优先级

| 编号 | 需求 | 说明 |
|------|------|------|
| F-009 | 拖拽调整优先级 | 通过拖动列表项调整事项优先顺序 |
| F-010 | 优先级持久化 | 排序结果保存，重启后保持 |

### 2.2 确认补充的功能需求

#### 2.2.1 任务基本字段（补充 #1）

| 编号 | 字段 | 必填 | 说明 |
|------|------|------|------|
| F-011 | 标题 | 是 | 一句话概括事项 |
| F-012 | 描述/备注 | 否 | 记录接口名、错误现象、复现步骤等 |
| F-013 | 创建时间 | 系统 | 创建时自动记录 |
| F-014 | 最后更新时间 | 系统 | 任何修改时自动更新 |

**示例**：

- 标题：`用户登录接口 500`
- 描述：`NullPointer at UserService:142，staging 环境`

#### 2.2.2 「待回复」上下文（补充 #2）

当状态为「待回复」时，支持填写以下可选字段：

| 编号 | 字段 | 说明 |
|------|------|------|
| F-015 | 等待对象 | 如「前端-张三」「产品-李四」 |
| F-016 | 等待原因 | 如「等接口文档」「等测试数据」 |

保存为「待回复」时，或已在「待回复」状态下修改等待对象/原因时，系统自动将待回复过程写入处置时间线（格式示例：`待回复：等待 联调方，原因 等待接口确认`）。

#### 2.2.3 筛选与视图（补充 #3）

| 编号 | 需求 | 说明 |
|------|------|------|
| F-017 | 按状态筛选 | 支持多状态组合筛选（如「未处置 + 待回复」） |
| F-018 | 按优先级排序 | 列表默认按拖拽设定的 priority 排序 |
| F-019 | 隐藏/显示已完成 | 默认隐藏已完成；可切换查看归档 |

#### 2.2.4 状态语义定义（补充 #4）

| 状态 | 含义 | 典型场景 |
|------|------|----------|
| 未处置 | 还没开始 | 新收到的需求 / bug |
| 处置中 | 进行中 | 正在写代码 / 排查 |
| 待回复 | 阻塞于外部 | 等联调方、产品、运维回复 |
| 已完成 | 彻底结束 | 已上线 / 已确认 |

#### 2.2.5 Deadline / 计划完成时间（补充 #8）

| 编号 | 需求 | 说明 |
|------|------|------|
| F-020 | 计划完成时间 | 可选字段，用户手动设置 |
| F-021 | 过期视觉提醒 | 已过期且未完成的事项在列表中高亮显示 |
| F-022 | 无推送通知 | MVP 不需要系统级提醒，仅列表内高亮 |

#### 2.2.6 处置过程管理（实现期补充）

| 编号 | 需求 | 说明 |
|------|------|------|
| F-023 | 编辑过程记录 | 可将已有过程内容加载到输入框并保存修改 |
| F-024 | 删除过程记录 | 单条过程可确认后删除 |

#### 2.2.7 桌面端 UI 约束（实现期补充）

JavaFX WebView 内 Vaadin 浮层组件（`Select`、`MultiSelectComboBox` 等）在 Dialog 中易失效，桌面端采用无浮层替代方案：

| 场景 | 原方案 | 现方案 |
|------|--------|--------|
| 编辑弹窗状态选择 | `Select` | `RadioButtonGroup` |
| 列表状态筛选 | `MultiSelectComboBox` | 各状态 `Checkbox` 组合 |

## 3. 非功能需求

| 编号 | 类别 | 需求 |
|------|------|------|
| NF-001 | 部署 | 本地桌面应用，双击即用，无需手动部署数据库或 Web 服务 |
| NF-002 | 数据 | 数据存储在本地 SQLite 文件，个人数据不出本机 |
| NF-003 | 性能 | 个人清单规模（数百条以内）下，操作响应即时 |
| NF-004 | 平台 | 优先支持 Windows 10+ |
| NF-005 | 启动 | 应用启动后自动打开工作窗口，用户无感知底层服务 |

## 6. 验收标准（MVP）

- [ ] 可创建、编辑、删除代办事项（标题必填）
- [ ] 列表展示状态、优先级、计划完成时间
- [ ] 拖拽调整优先级并持久化
- [ ] 四种状态可切换，语义符合定义（「处置中」表示进行中）
- [ ] 「待回复」状态可填写等待对象和等待原因，保存后自动写入处置过程时间线
- [ ] 可追加、编辑、删除处置过程记录，按时间倒序展示
- [ ] 支持按状态筛选、隐藏/显示已完成
- [ ] 过期未完成事项有明显视觉高亮
- [ ] 打包为 Windows 可执行程序，双击启动
